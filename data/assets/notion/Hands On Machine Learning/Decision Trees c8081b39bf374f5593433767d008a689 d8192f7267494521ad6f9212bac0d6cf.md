- Explain the difference between white box models vs black box models?
    - White box models makes decisions that are easy to interpret and understand how the model came to that interpretation
    - Black box models makes decision that are hard to understand why it came to that interpretation
- Explain how Decision Trees can make class probabilities?
    - First it traverses the tree to find the leaf node for the instance
    - returns the ratio of training instances of the class divided by the sample size
- Explain the classification and regression tree (CART) Algorithm?
    - The CART algorithm is a greedy algorithm used to find an optimum way of training a decision tree by recursively splitting the training set into subsets based on features and thresholds until the max depth is reached or the algorithm cannot find a split that reduces impurity
    1. Splits the training set into two subsets using a single feature and a threshold
        1. Chooses the feature and threshold by finding the purest subset which is weighted by their size
    2. Splits subsets using the same logic recursively until the max depth is reached or it cannot find a split that reduces impurity
    - The CART algorithm for regression works the same way but instead of minimizing the gini score or impurity it minimized the cost function of the algorithm
- What is the time complexity of making predictions for Decision Trees?
    - O(log2(m))
    - Since Decision Trees are usually balanced, traversing through them requires you to only check the value of one feature
- Why is a Decision Tree and nonparametric model?
    - A nonparametric model is a model where the number of parameters is not determined prior to training
    - Decision Trees make few assumptions about the training data allowing it to easily overfit the data
- How can we restrict a Decisions Tree degree of freedom?
    - We can use the following hyper parameters to regularize the model
    - Increasing the min_* or reducing the max_* hyper parameters will regularize the model
    - max_depth - How many levels the tree can have
    - min_samples_split - minimum number of samples a node must have before it can be split
    - min_samples_leaf - minimum number of sample a leaf node must have
    - min_weight_fraction_leaf - same has min_samples_leaf but expressed as a fraction
    - max_leaf_nodes - max amount of leaf nodes the model can have
    - max_features - max number of features that are evaluated for splitting at each node
- What are some limitations of Decision Trees?
    - They love right angels and tend to fit the data weirdly if needed
    - Very sensitive to small variations within the data so needs to be regularized
