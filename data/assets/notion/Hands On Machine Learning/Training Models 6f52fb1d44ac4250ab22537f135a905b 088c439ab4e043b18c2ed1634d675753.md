- How do you find a good learning rate for Gradient Descent?
    - Use GridSearch
    - limit the number of iterations so that Grid Search can eliminate models that take too long
- Explain Stochastic Gradient Descent
    - Picks a random instance in the training set and computes the gradient based on that single instance which causes the algorithm to be faster and use less memory
    - Algorithm is not regular and bounces around often, final answer will be good but not optimal
    - irregular nature helps jump out of a local minima and has a better chance of finding global minimum
    - Steps
        1. Picks a random instance to compute gradient
        2. irregularly tries to minimize the cost function
        3. Learning Schedule helps tweak the learning rate to help algorithm settle
- Explain Mini-batch Gradient Descent
    - computes gradients on a small random set of instances from the full training sets
    - Less erratic than Stochastic GD but takes less time then Batch GD
    - Gets the performance boosts from hardware optimization of matrix operations when using GPUs
- Explain Polynomial Regression
    - Add powers of each feature as a new feature and then train a linear model on this extended set of features
    - Just like how polynomial functions has a lot of curves and ups and downs, it can be used to fit data that is not represented in a straight line
- Explain Learning Curves
    - Plots of the model’s performances on the training and validation sets used to determine if the model is under-fitting or overfitting the data
    - To make a learning curve make a plot with the cost function for both the training set and validation set
    - If both of the cost functions for the validation and training set is bad then the model is under fitting
        - Using a more complex model can help
        - Coming up with better features can help
    - If there is a big gap between the cost functions of the validation and training sets then the model is over fitting
        - Adding more data can help
- How are three ways you can express a model’s generalization error?
    1. Bias - wrong assumptions about the data such as assuming the data is linear when it is quadratic
    2. Variance - How sensitive the model is to small variations in the training data
    3. Irreducible error - The noisiness of the data itself
- Explain the Bias/Variance Tradeoff
    - When the model has a high-bias it is likely to under fit the data because the assumptions about the data is wrong
    - When the model has many degrees of freedom it likely has high variance and is prone to overfitting the training data
    - Increasing the models complexity you will increase its variance and reduce its bias
    - Decreasing the models complexity you will decrease its variance and increase its bias
- Explain Ridge Regression, Lasso Regression, and Elastic Net and when you should use them
    - Explain Ridge Regression
        - A regulated version of linear regression that focuses on fitting the data and keeping the wights as small as possible to prevent overfitting the data
        - This is done by adding a (a hyperparameter) to the cost function when training the model which controls how much you want to regularize the weights
        - When a = 0, it is just linear regression and the higher the a is the more the line just go through the mean of the data meaning all the weights are equal to 0
    - Explain Lasso Regression
        - Another regularized version of of Linear Regression dome by adding a regularization term to the cost function but uses the l1 norm of the weight vector instead of half the square l2 norm
        - Tends to set the least important features weights to zero
    - Elastic Net
        - A mix of Ridge and Lasso Regression where you take control of the regularization terms by a mix ratio (r)
        - When r = 0 it is just Ridge Regression
        - When r = 1 it is Lasso Regression
    - When to use?
        - Almost always preferable to have a little bit of regularization when using linear regression
        - If you think only a few features are helpful them use Lasso or Elastic Net since they minimize the useless features weights
        - Ridge Regression is a good default
- Explain early stopping
    - A way to regularize iterative learning algorithms such as Gradient Descent by stopping the training as soon as the validation error reaches a minimum
    - Naturally, as the epochs go by, the prediction error on the training set and validation set goes down. When the validation error starts to increase that means the model is overfitting to the training data.
- Explain Logistic Regression
    - A binary classifier that estimates the probability that an instance belongs to a certain class and if that probability is 0.5 or greater it predicts that the instance belongs to that class
    - No Normal Equation for Logistic Regression and will have to use an optimization algorithm
- Explain Softmax Regression
    - logistic regression that is generalized to support multiple class directly by computing a score for each class and then estimating the probability of each class by applying the softmax function
    - Variables need to be mutually exclusive (cannot happen at the same time)
